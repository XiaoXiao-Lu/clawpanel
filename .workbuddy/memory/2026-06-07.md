# 2026-06-07 UI Redesign 工作记录

## 工作内容
对 ClawPanel 项目进行了全面的 UI redesign，解决界面风格不统一、布局不合理、交互体验差、观感廉价的问题。

## 核心问题诊断
1. **polish.css 是最大毒瘤** — 用 60+ 选择器列表 + `!important` 强制给所有组件加渐变顶边、强阴影、杂色背景
2. **过度装饰** — 每个卡片顶部都有 indigo/orange 渐变线，hover 统一上移+阴影放大
3. **背景杂乱** — 多层径向渐变叠加，没有视觉重点
4. **颜色混乱** — indigo + orange 强制组合 everywhere，没有主次之分
5. **阴影过重** — 多层 box-shadow 让界面显得脏
6. **布局缺乏呼吸感** — 内容拥挤，间距不一致

## 已完成的改进

### 1. 设计系统重构 (`variables.css`)
- 建立了新的克制型色彩系统（降低饱和度、减少杂色）
- 简化了阴影层级（3 层 → 柔和有层次）
- 保留了完整的旧变量兼容层，确保所有页面样式无缝适配

### 2. 全局布局重构 (`layout.css`)
- 侧边栏：去掉复杂渐变背景，改为干净的纯色
- 导航项 active 状态：从渐变条改为柔和的背景色+品牌色文字
- 页面标题：去掉底部的渐变装饰线
- 横幅：简化 update-banner 和 gw-banner 的样式

### 3. 组件样式重构 (`components.css`)
- 按钮：去掉渐变背景，改为纯色 + 微妙的 hover 阴影
- 卡片：去掉顶部渐变线，改为干净的边框 + 柔和阴影
- 表单输入框：简化 focus 状态，去掉过度发光
- Toast：去掉全色文字，改为左侧色条 + 正常文字
- Modal：简化阴影和边框
- Toggle Switch：修复了暗色模式下 checked 状态的红色 bug

### 4. 润色层大幅简化 (`pages/polish.css`)
- 从 540+ 行、60+ 选择器、大量 `!important` 简化为 200+ 行的轻量覆盖
- 不再强制给所有卡片加渐变顶边
- 不再强制混色所有边框和背景
- 保留引擎特定的差异化样式但大幅简化

### 5. 页面样式清理
- `dashboard.css`：去掉 health-card 的渐变背景和 4px 粗左边框
- `settings.css`：去掉 config-section 的渐变顶边和渐变背景
- `services.css`：去掉 service-card 的渐变顶边和 hover 上移
- 批量清理了 assistant.css / channels.css / services.css / settings.css 中的 `transform: translateY(-1px)` hover 效果

## 构建验证
- `npm run build` 通过，无 CSS/JS 错误
- CSS 体积有所优化

2. 进一步审查各页面组件的 DOM 结构，统一卡片和列表的 HTML 模式
3. 考虑引入 CSS 预处理器（如 PostCSS）来更好地管理设计系统
