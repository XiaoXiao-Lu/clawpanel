## ClawPanel UI 设计原型

为 OpenClaw/ClawPanel 项目创建了完整的 UI 设计原型文件 `ui-design-prototype.html`。

### 设计系统亮点
- **深色主题为主**：基于 deep space 玻璃拟态风格，背景层 #080a0e → #0e1218 → #111520
- **品牌色**：Indigo oklch(55% 0.22 270) + Lobster Orange oklch(65% 0.18 55) 双品牌色渐变
- **字体**：Plus Jakarta Sans (展示/正文) + JetBrains Mono (代码) + Noto Sans SC (中文)
- **莫兰迪色系侧边栏图标**：为每个导航项分配独特的柔和色彩
- **微动效**：页面进入动画、卡片 hover 上浮、状态点脉冲、进度条填充、柱状图生长
- **WCAG AA 无障碍**：focus-visible 环、减少动画偏好支持、44px 最小触控目标

### 包含页面（10个）
1. 仪表盘 - 6个统计卡片 + 命令中心 + 健康度进度条 + 日志查看器
2. AI 助手 - 4种操作模式切换 + 8个技能卡片
3. 实时聊天 - 流式对话界面 + 模型选择 + 多模态输入
4. 服务管理 - Gateway 启停控制 + 版本信息 + 备份配置
5. 模型配置 - 数据表格 + 标签过滤 + 批量测试
6. Agent 管理 - 3列卡片网格 + 身份配置 + 工作区标签
7. 记忆管理 - 4个核心文件卡片 + Markdown 预览
8. 网关配置 - 基础设置 + Tailscale 组网
9. 消息渠道 - 4个平台卡片（飞书/Telegram/Discord/QQ）
10. 使用情况 - 4个指标卡片 + 柱状图 + 模型排行进度条
11. 日志查看 - 带颜色区分的日志级别 + 标签过滤
12. 技能管理 - 技能卡片 + 开关控制

### 技术实现
- 纯 HTML/CSS/JS，零框架依赖
- CSS 变量设计令牌系统（暗/亮双主题）
- 响应式布局（桌面/平板/手机）

### 实际修复项

**2. Dashboard 排版优化**
- stat-card 数值字号：`font-size-2xl` → `font-size-xl`（避免过大）
- stat-cards 间距：`--space-lg` → `--space-md`（更紧凑）

- health-card 添加 ≤860px 断点（垂直堆叠）
- 页面标题区增加底部边框分隔线

**3. Services 服务管理重构为 Tab 切换结构（P0）**
- **彻底废弃原有的左右双栏网格布局**：原布局不仅容易产生挤压，而且对于大屏来说右边栏太高、小屏下又会变成上下两个超长纵向块，体验极差
- **引入 Tab 选项卡导航**：
  - `运行服务` (Running Services)：包含版本检测卡片、服务网关列表、Docker 多实例管理
  - `配置文件` (Config File)：包含 openclaw.json 的在线文本域编辑器、配置校准继承/重置卡片
  - `备份管理` (Backups)：包含 openclaw.json 的防灾备份列表、创建/恢复操作
- **布局流式优化**：
  - 每一个 Tab 采用单栏流式卡片容器 (`max-width: 860px; margin: 0 auto`)，整体居中对称，再无左右宽度不合理挤压
  - 将配置文件文本编辑器的高度扩展至 `min-height: 380px`，大字号输入更清晰
  - 在 `services.js` 绑定事件并支持 i18n 自适应 Tab 标签切换，过渡动效更平滑

**4. 修复聊天顶部栏 (Chat Header) 挤压断裂错误（P0）**
- **问题定位**：从截图可以看出，当宽度变小时（或者开启 Workspace 面板时），顶部栏（`.chat-header`）的子元素没有弹性，搜索栏与下拉选择框出现部分挤压、甚至重叠断裂
- **响应式修复**：
  - 针对 `max-width: 760px`（小屏及开启工作区挤压状态）添加完整的弹性折行和对齐支持
  - `.chat-header` 和 `.chat-header-actions` 改为 `flex-wrap: wrap` 并补充 `gap: var(--space-xs)`，避免元素硬挤到视口外
  - 搜索栏 `.chat-message-search` 在窄屏下扩展为 `width: 100%`，其内部 `input` 改为 `flex: 1` 自适应，保证即使缩到最窄输入框也清晰可见且可点击
  - 状态指示字号和容器优化，解决在屏幕变窄时遮挡搜索框/重叠的问题

**5. 修复移动端下实时聊天底部输入框显示不全/视口溢出错误（P0）**
- **问题定位**：在宽度变小（例如移动端、小窗口、或打开 Workspace 挤压）的情况下，主容器没有被正确限制为视口高度 (`100%` / `100vh`) 且产生了页面级别的 `overflow: auto`。这会导致底部的 `.chat-input-area` 被顶出屏幕外，无法完整显示
- **深度修复**：
  - 在 `polish.css` 移动端媒体查询（`max-width: 768px`）下，显式将 `.chat-page` 和 `.ast-page` 的高度设为 `height: 100% !important; overflow: hidden`，从顶层断绝滚动条溢出
  - 显式将聊天主容器 `.chat-main` 和 `.ast-main` 的高度设为 `height: 100% !important; display: flex; flex-direction: column; overflow: hidden`
  - 这样主容器内部的 `.chat-messages` 会正确继承 `flex: 1; overflow-y: auto` 负责内部消息滚动，而底部的 `.chat-input-area` 能够百分百贴在屏幕底部，显示完整，无论虚拟键盘或视口如何收缩都绝不溢出

**6. 支持晴辰助手会话列表默认关闭与状态持久化（P1）**
- **问题定位**：晴辰助手页面的会话列表（`.ast-sidebar`）原先通过 `max-width: 980px` 检测强制在大屏下默认打开。当用户打开该页面时，列表会直接挤压核心聊天区，影响沉浸感，且每次跳转后状态都会重置。
- **功能修复**：
  - 在 `assistant.js` 页面渲染初始化逻辑中，引入 `localStorage` 持久化状态，键名为 `clawpanel-assistant-sidebar`
  - 首次访问时：若处于窄屏环境（`≤980px`）默认不加载 `open` 类名（即默认折叠隐藏）；若为宽屏则默认展开
  - 之后每次用户点击折叠按钮 `#ast-btn-toggle` 切换侧边栏时，自动同步新的状态值（`true` / `false`）到 `localStorage` 中
  - 重新加载页面或切换路由回来时，会完美记住并恢复上一次折叠/打开的状态，无需每次重复折叠
