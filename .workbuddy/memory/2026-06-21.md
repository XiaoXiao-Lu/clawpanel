## Markdown 代码块复制+折叠按钮

### 问题
- 晴辰助手（assistant/chat 页面）的 Markdown 代码块没有复制和折叠按钮
- `src/lib/markdown.js` 虽然生成了 `<button class="code-copy-btn">` HTML，但整个项目没有对应 CSS 样式，按钮不可见
- `src/engines/hermes/lib/markdown-renderer.js` 完全没有复制/折叠功能
- 两个渲染器都缺少代码块折叠功能

### 修复内容
1. **代码块 HTML 结构改造**：从 `<pre>` 直接包含按钮改为 `<div class="code-block-wrapper">` 包裹 header + pre 结构
   - header 含语言标签 `.code-lang`、复制按钮 `.code-copy-btn`、折叠按钮 `.code-fold-btn`
   - 超过 15 行的代码块才显示折叠按钮（`data-foldable` 属性）
2. **CSS 样式**：在 `polish.css` 添加统一的 `.code-block-wrapper` 工具栏样式
   - 用 `.code-block-wrapper pre.code-block-pre`（0-3-1 优先级）覆盖各页面已有的 pre 样式
   - 折叠按钮用 CSS `::after` 伪元素画 ▾/▸ 箭头
   - 折叠状态 `.collapsed` 隐藏 `<pre>`
3. **事件委托**：用 `window.__codeBlockEventsReady` 全局标志防止 markdown.js 和 hermes markdown-renderer.js 重复注册
4. **DOMPurify 配置更新**：ALLOWED_ATTR 添加 `data-fold-btn`、`data-foldable`、`title`
5. **复制按钮文案**：从 "Copy" 改为 "复制"

### 修改文件
- `src/lib/markdown.js` — 代码块结构、DOMPurify 配置、事件委托
- `src/engines/hermes/lib/markdown-renderer.js` — 代码块结构、事件委托
- `src/style/pages/polish.css` — 工具栏 CSS 样式
- `tests/markdown-renderer.test.js` — 更新代码块断言

### 验证

- JS 模块加载无错误
