# 模型配置页 UI/UX 优化设计

> 日期：2026-06-10
> 状态：Draft
> 项目：ChatCraw (ClawPanel v0.18.x)
> 核心问题：信息密度过高，视觉层级不清晰

---

## 一、问题分析

### 1.1 现状

模型配置页 (`src/pages/models.js`, 2486 行) 采用左右分栏布局：
- **左侧 (250px, sticky)**：控制台 — 主模型 Combobox + 备选链 + 路由预设 + 推理开关 + 并发数 + Gateway 状态 + 备选瀑布流入口，6 个功能区纵向堆叠
- **右侧**：服务商 Tab + 搜索 + 模型卡片网格

### 1.2 核心问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | 控制台 6 个功能区挤在 250px 宽、间距 0-4px | 视觉密度极高，用户无法快速定位所需操作 |
| 2 | 统计栏、控制台、卡片网格三层无清晰分隔 | 信息层次模糊，所有内容等权呈现 |
| 3 | 模型卡片内字段等权：名称+ID+类型+状态+按钮+徽章全部平铺 | 主模型和普通模型视觉权重相同，核心信息不突出 |
| 4 | 卡片间距紧凑，横向 3 列填满 | 无呼吸感，密集排列增加认知负荷 |
| 5 | Qtcool 推广横幅常驻顶部 | 非核心信息占据黄金位置 |

### 1.3 设计目标

- **降密度**：功能区之间有明确留白，卡片间距从 8px → 16px
- **建层级**：主模型 > 备选 > 普通，三档视觉权重
- **分区域**：Hero / 概览 / 服务商，三层纵向分区
- **不改功能**：所有现有功能保留，只调整布局和视觉

## 二、信息架构重组

### 2.1 从左右分栏 → 纵向三区

```
┌──────────────────────────────────────────────┐
│ Hero 区：主模型 + 备选链                      │  ← 用户第一眼看到
├──────────────────────────────────────────────┤
│ 概览条：统计 + 路由预设 + 推理开关            │  ← 一行紧凑
├──────────────────────────────────────────────┤
│ 服务商 & 模型                                 │
│   Tab + 搜索 + 操作                           │
│   模型卡片网格 (主次分明)                     │  ← 核心操作区，面积最大
│   批量操作 / 备选瀑布流 (展开)                │
└──────────────────────────────────────────────┘
```

### 2.2 各区域详细设计

#### 区域 1：Hero — 主模型 + 备选链

**布局**：占满页面宽度，内部分左右两栏

- **左侧**：主模型大卡片
  - 绿色边框 (1px)，圆角 12px
  - "P" 图标 + 模型名 (14px/500) + "主模型" 标签 (绿色徽章)
  - 第二行：服务商 · API类型 · 连通状态
  - 第三行：延迟 · 上下文长度 (小字灰色)
  - 右侧：测试连通按钮 (实心绿色)
  - 点击卡片：展开主模型切换 Combobox

- **右侧**：备选链
  - 标题 "备选链" + "管理" 链接
  - 横排药丸：claude → deepseek → + 添加
  - 每个药丸：蓝色边框 + 优先级编号
  - 点击药丸：弹出备选编辑弹窗
  - 点击 "管理"：展开瀑布流编辑器

**视觉特征**：
- Hero 区背景：`color-mix(in srgb, var(--success) 4%, var(--bg-primary))`
- 区域下方间距：24px

#### 区域 2：概览条 — 统计 + 快捷操作

**布局**：一行横向排列，左右对齐

- 左侧：3 个统计胶囊 (服务商数 / 模型数 / 网关状态)
- 右侧：路由预设按钮组 (快捷 / 稳定 / 自定义) + 推理开关 + 并发数
- 背景：白色，0.5px 边框，圆角 8px
- 区域下方间距：20px

#### 区域 3：服务商 & 模型

**布局**：占满页面宽度，占最大面积

- 顶栏：服务商 Tab + 搜索框 + "添加模型" / "添加服务商" 按钮
- 卡片网格：`grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))`
- 卡片间距：`gap: 16px`（从 8px 提升）
- 区域下方：批量操作区 + 备选瀑布流 (默认折叠)

### 2.3 Qtcool 推广横幅处理

- 从常驻顶部 → 移到服务商区上方，可折叠
- 或改为：概览条中一个小的 "晴辰云" 链接，点击展开推荐

## 三、模型卡片视觉层级

### 3.1 三档视觉权重

| 角色 | 边框 | 高度 | 信息量 | 额外 |
|------|------|------|--------|------|
| 主模型 | 1px solid `var(--success)` | 72px | 名称+服务商+类型+状态+延迟+ctx | "主模型" 绿色标签 |
| 备选模型 | 0.5px solid `var(--info)` | 64px | 名称+服务商+类型+状态 | 优先级编号 + "备选" 蓝色标签 |
| 普通模型 | 0.5px solid `var(--border)` | 52px | 名称+服务商+状态 | 可选：推理/视觉等小标签 |

### 3.2 卡片内部布局

```
┌─────────────────────────────────────┐
│  [P]  gpt-4o              [主模型]  │  ← 第一行：图标+名称+角色标签
│  openai · Chat · 已连通             │  ← 第二行：服务商·类型·状态
│  延迟 1.2s · 200K ctx    [测试]    │  ← 第三行(仅主模型/备选)
└─────────────────────────────────────┘
```

- 操作按钮（编辑、删除、设为主模型）→ hover 时显示
- 推理模型标签 → 紫色小药丸
- 连通状态 → 绿点/红点 + 文字

### 3.3 卡片间距

- `gap: 16px` (从 8px)
- 卡片内边距：`padding: 14px 16px` (从 10px 12px)

## 五、JS 改动清单

### 5.1 render() 入口

- 移除左右分栏的 DOM 结构 (`models-workbench`)
- 改为三区纵向：`models-hero` → `models-overview-bar` → `models-provider-section`

### 5.2 renderConsole() 重构 → 拆分为

- `renderHero()`：主模型大卡片 + 备选链横排
- `renderOverviewBar()`：统计胶囊 + 路由预设 + 推理开关 + 并发数
- 保留原有事件绑定逻辑，只改 DOM 结构

### 5.3 renderProviders() 微调

- 卡片渲染增加 `--primary` / `--fallback` / `--default` 类
- 操作按钮改为 hover 显示
- 卡片间距由 CSS grid gap 控制

### 5.4 主模型切换交互

- 点击 Hero 区主模型卡片 → 展开内嵌 Combobox（非弹窗）
- 选择后自动收起，显示新主模型信息
- 保留原有 Combobox 组件，只改变触发方式

- 自动保存 & 撤销机制

- 远程模型列表获取
- 晴辰云集成
- Gateway 应用
- 备选瀑布流编辑器（只改触发入口，不改功能）
