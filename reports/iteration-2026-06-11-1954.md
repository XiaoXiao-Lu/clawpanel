## 目标
- 继续优化专家团执行过程中的 UI/UX。

## 本轮改动
- `src/pages/assistant.js`
  - 为专家团实时输出 slot 增加隐藏的 `sr-only` polite/atomic live announcer。
  - 新增 3.5 秒节流的正文快照播报，避免 token 级 delta 频繁打断读屏。
  - DOM 增量更新不再整块替换实时 slot，而是复用/确保稳定的 announcer 节点，只更新可视正文和节流后的播报文本。
  - 保留尾部跟随滚动：实时正文更新后继续滚到底部。
  - cleanup 时清理专家团 DOM payload 和播报节流状态，避免跨页面残留。
- `src/style/assistant.css`
  - 将 `.ast-expert-live-text` 从 `overflow: hidden` 改为 `overflow-y: auto`。
  - 移除底部 `mask-image` 淡出，改用细滚动条，方便用户回看实时控制台正文。
- `tests/expert-team-live-ux.test.js`
  - 扩展专家团实时 UI/UX 回归覆盖：尾部模式、自动滚动、概览 live region、低频正文播报、稳定 announcer 节点、控制台可滚动回看。

## 二次全局复盘
- 复查了本轮 diff，范围限定在专家团执行 UI 的实时播报与回看体验，没有触碰执行引擎和专家团配置模型。
- 低频读屏播报使用稳定 DOM 节点和 3.5 秒节流，避免每个 delta 都进入 live region。
- DOM debounce 期间保留 pending announcement，避免首个节流快照被后续空播报覆盖。
- 控制台正文改为可滚动后，实时尾部跟随仍由 `scrollTop = scrollHeight` 保持。

## 风险
- 当前测试主要是静态源码断言 + 现有专家团浏览器 smoke；真实读屏行为仍建议后续用 NVDA/VoiceOver 做人工抽检。
- 3.5 秒播报间隔是保守默认值，不同读屏器和生成速度下可能需要微调。
- `.ast-expert-live-text` 全局从隐藏改为滚动，视觉行为更可回看，但可能让部分长正文区域出现滚动条。
