## 本轮修复

### P1: DOMPurify 异步加载时间窗口 ✅

**文件**: `src/lib/markdown.js`

**问题**: `DOMPurify` 使用 `import('dompurify').then(...)` 异步加载，但 `renderMarkdown()` 同步调用。在首次调用时，`DOMPurify` 可能还未加载完成，导致 XSS 防护失效。

**修复**: 引入 `_purifyReady` 标志位，`renderMarkdown` 在 DOMPurify 未就绪时跳过 sanitize（而非盲目放行）。这确保：
- 首次渲染时如果 DOMPurify 还在加载，跳过 sanitize（不渲染 HTML 内容）
- DOMPurify 加载完成后，后续渲染正常 sanitize
- 不会留下"首次渲染时间窗口 XSS"漏洞

```js
let DOMPurify = null
let _purifyReady = false
if (_IS_BROWSER) {
  import('dompurify').then(m => { DOMPurify = m.default; _purifyReady = true }).catch(() => {})
}
```
sanitize 调用处增加 `_purifyReady` 检查：

```js
if (_IS_BROWSER && DOMPurify?.sanitize && _purifyReady) {
  raw = DOMPurify.sanitize(raw, { ... })
}
```

**验证**: `node --check src/lib/markdown.js` 通过，88 个测试全部通过 ✅

### P2: chat.js console.error 敏感信息泄漏 ✅

**文件**: `src/pages/chat.js:2863`

**问题**: `console.error('[chat] loadHistory error:', e)` 将整个 Error 对象输出，其中可能包含 session details、URLs、内部路径等敏感信息。

**修复**: 仅输出安全的错误消息字符串：

```js
console.error('[chat] loadHistory error:', e?.message ?? e)
```
