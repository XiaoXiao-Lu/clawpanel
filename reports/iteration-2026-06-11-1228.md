# Iteration 2026-06-11 12:28

## 目标
- 加固 headless 静态文件服务的 HTTP 方法语义。
- 让静态资源明确只接受 `GET`/`HEAD`，对不应落入 SPA fallback 的写入类方法返回 `405`。

## 地毯式审查摘要
- 当前分支为 `main`，领先 `origin/main` 19 个本地提交；工作区仍有大量既有未提交改动，集中在专家团、模型页、助手页、命令面板、多语言、样式和 `.workbuddy`。
- 基线 `npm test` 通过 543/543，`npm run build` 通过。
- 最近报告建议继续关注专家团键盘可达性、模型页 combobox 和 headless server 行为；专家团和模型页生产文件已有既有脏改，本轮选择干净的 `scripts/serve.js` 继续补静态服务边界。
- 审查发现 `serveStatic` 只按路径处理请求，未区分 `GET`/`HEAD`/`POST`；这会让 POST 等写入语义请求进入静态文件或 SPA fallback。

## 修改
- `scripts/serve.js`
  - `serveStatic` 仅允许 `GET` 和 `HEAD`。
  - 其它方法返回 `405 Method Not Allowed`，并设置 `Allow: GET, HEAD`。
  - `HEAD` 请求复用文件解析、缓存头和 MIME 设置，但不返回 body。
  - 错误响应和 404 在 `HEAD` 下也保持无 body。
- `tests/serve-static-security.test.js`
  - `readResponse` 支持指定 HTTP method，并记录 `Allow` 响应头。
  - 增加 `HEAD` 无 body、保留缓存头和 content-type 的测试。
  - 增加 `POST` 返回 405 且不进入静态文件读取的测试。
