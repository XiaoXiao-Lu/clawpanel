# Iteration 2026-06-11 (专家团执行 UI/UX 增强)

## 目标
- 基于上一轮对专家团执行 UI/UX 的分析，落地最高价值的两项改进：
  1. 实时进度对读屏可见（aria-live）。
  2. 流式预览跟随最新生成的尾部并自动滚动，增强“正在生成”的实时感。

## 地毯式审查摘要
- 起始 HEAD 9162cb85，worktree 干净。基线 npm test 561/561、npm run build 通过。
- 复读 assistant.js 专家团执行渲染链（约 70 个函数）与 expert-team-runner.js 事件流。
- 确认上一轮分析中两个最值得修的点：
  - 执行卡片没有任何 aria-live 区域，读屏用户无法感知阶段/意见数/异常/综合/交付的实时变化。
  - expertTeamLiveText 只从头部截断（slice(0,limit)），长输出时实时面板显示开头而非最新尾部，且没有滚到底，实时感打折扣。

## 修改
- src/pages/assistant.js：
  - expertTeamLiveText 增加 opts.tail 选项：tail 模式从尾部切片、在最近句界对齐、前缀省略号，跟随最新输出；默认（head）模式保持不变，供已完成/综合摘要使用。
  - 三处“正在输出”的实时调用改为 tail 模式：ex-live-text 主面板（900）、ast-expert-live-text 控制台（700/900）、节流 DOM 更新 payload（scheduleExpertTeamLiveDomUpdate）。已完成/done/摘要预览（180/560/1200 等）仍用 head 模式。
  - applyExpertTeamLiveDomUpdate 在写入后把可滚动的 .ex-live-text/.ast-expert-live-text 滚到底（scrollTop=scrollHeight）。
  - ex-overview 概览条加 role="status" aria-live="polite" aria-atomic="true"：概览只在非 delta 渲染时重建，因此按 polite 播报阶段/意见数/异常/耗时变化，避免逐 token 刷屏。
- 新增 tests/expert-team-live-ux.test.js：断言 tail 选项与尾部切片逻辑、三处以上 tail 调用、自动滚到底、概览 polite atomic live region。

## 风险
