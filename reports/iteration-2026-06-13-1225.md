### P0: skills.js 图标后备逻辑断裂

**文件**: [skills.js:288-295](src/pages/skills.js#L288-L295)

**问题**: 用户改动移除了内联 `onerror` 属性（安全改进），但没有提供替代实现：
```html
<!-- 用户改动后 — onerror 被移除，后备逻辑完全失效 -->
<img src="${esc(primary)}" alt="" loading="lazy"
  data-icon-fallbacks="${esc(fallbacks)}" data-icon-fallback-icon>
```

**修复**: 在 `bindEvents` 开头添加事件委托处理图片加载失败：
```javascript
page.addEventListener('error', (e) => {
  const img = e.target.closest('[data-icon-fallback-icon]')
  if (!img) return
  const fallbacks = JSON.parse(img.dataset.iconFallbacks || '[]')
  const next = fallbacks.shift()
  if (next) {
    img.dataset.iconFallbacks = JSON.stringify(fallbacks)
    img.src = next
  } else {
    img.style.display = 'none'
  }
}, true)
```

**效果**: 保持安全改进（无内联脚本），同时恢复图标后备加载功能。

### P1: sidebar.js catch 参数变量名错误

**文件**: [sidebar.js:302](src/components/sidebar.js#L302), [sidebar.js:370](src/components/sidebar.js#L370)

**问题**: 用户改动把 `err` 改成了 `e`，导致 catch 块中使用未定义的 `e`：
```javascript
// 之前 (正确): .catch(err => { console.error(..., err?.message ?? err) })
// 用户改动后:  .catch(err => { console.error(..., e?.message ?? e) }) ← ReferenceError
```

**修复**: 恢复正确的变量名 `err`：
```javascript
.catch(err => { console.error('[sidebar] ...', err?.message ?? err) })
```

**效果**: 修复内核升级和引擎切换失败时的日志输出，避免 ReferenceError。
