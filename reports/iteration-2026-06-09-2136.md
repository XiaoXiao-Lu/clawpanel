- 初始工作区已有大量既有脏改，包含 `.workbuddy`、`index.html`、多份全局样式和未跟踪设计系统报告；本轮均视为既有工作，未回滚、未纳入提交。

- 基线 `npm run build` 通过；基线 `npm test` 失败在 `tests/design-system-polish.test.js`，原因是测试仍断言旧 Precision Noir token/焦点契约，同时旧页面继续引用未定义的 `--text-xxs`。

## 修改说明
- `src/style/compat.css`
  - 新增运行时兼容层，定义 `--text-xxs: var(--text-2xs)`，让旧 Agents/Assistant/Dashboard/About 样式在 Aether Core v2 下继续解析。
- `src/main.js`
  - 在 `variables.css` 后导入 `compat.css`，确保兼容 token 不是只存在于测试聚合文件里。
- `tests/design-system-polish.test.js`
  - 将圆角、信息边框和焦点断言更新为 Aether Core v2 当前契约。
  - 增加主入口导入 `compat.css` 的断言，防止运行时兼容层丢失。
- `src/pages/assistant.js`
  - 补充 `escapeAttr as escAttr` 导入，修复助手页专家团渲染时 `escAttr is not defined` 的运行时错误。
- `tests/expert-teams-config.test.js`
  - 增加助手专家团入口必须导入 `escAttr` 的回归断言。
