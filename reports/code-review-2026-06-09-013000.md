## 已修复项目 (15项)

### 🔴 必须修改 (已完成)

**1. Toast 定位与桌面标题栏冲突**
- **文件**: `src/style/components.css`
- **问题**: `.toast-container` 固定 `top: 56px`，桌面端有 36px titlebar 时 toast 被遮挡
- **修复**: 改为 `top: var(--space-4)` + `body.has-desktop-chrome .toast-container { top: calc(36px + var(--space-4)) }`

**2. Focus-visible 双重轮廓冲突**
- **文件**: `src/style/reset.css`
- **问题**: 全局 `outline: none; box-shadow: var(--shadow-glow)` 与 `.btn:focus-visible` 的 `outline: 2px solid` 冲突，部分元素焦点不可见
- **修复**: 统一为 `outline: 2px solid var(--brand); outline-offset: 2px`，去掉 `box-shadow` 覆盖

**3. .chat-attach-btn 重复定义**
- **文件**: `src/style/chat.css`
- **问题**: 同一选择器出现两次（第 956 行和第 1414 行），后定义覆盖前定义的完整样式，导致按钮尺寸/边框/背景丢失
- **修复**: 删除重复定义，保留完整样式版本，按钮尺寸统一为 44×44

**4. 消息图片尺寸过小**
- **文件**: `src/style/chat.css`
- **问题**: `.msg-img { max-width: 200px; max-height: 200px }` 在桌面端太小
- **修复**: 改为 `max-width: min(280px, 60vw); max-height: 280px`，增加 hover 放大效果

### 🟡 建议优化 (已完成)

**5. 移动端侧边栏缺少 ESC 键支持**
- **文件**: `src/components/sidebar.js`
- **问题**: 移动端侧边栏只能点击 overlay 或 X 按钮关闭，无键盘退出路径
- **修复**: 添加全局 `keydown` 监听器，ESC 键关闭打开的移动端侧边栏

**6. 聊天侧边栏浮层模式缺少 ESC 键支持**
- **文件**: `src/pages/chat.js`
- **问题**: ≤1200px 时侧边栏变为浮层，但无 ESC 退出
- **修复**: 添加页面级 `keydown` 监听器

**7. 导航分区折叠缺少 ARIA 属性**
- **文件**: `src/components/sidebar.js`
- **问题**: `.nav-section-title` 有 `data-toggle` 但无 `role="button"`、`tabindex="0"`、`aria-expanded`
- **修复**: 添加完整 ARIA 标记，键盘 Enter/Space 可触发折叠

**8. 魔法数字 z-index**
- **文件**: `src/style/pages/services.css`
- **问题**: `.upgrade-task-bar { z-index: 9999 }` 和 `.docker-dialog-overlay { z-index: 1000 }` 使用硬编码数字
- **修复**: 统一使用 CSS 变量 `var(--z-max)` 和 `var(--z-modal)`

**9. 命令面板移动端适配**
- **文件**: `src/components/command-palette.css`
- **问题**: 固定 520px 宽度在小屏溢出
- **修复**: 添加 `@media (max-width: 600px)` 使全宽显示，输入框 font-size 16px 防止 iOS 自动缩放

**10. 聊天输入框 iOS 自动缩放**
- **文件**: `src/style/chat.css`
- **问题**: `font-size: var(--font-size-md)` (14px) 在 iOS Safari 触发自动缩放
- **修复**: 改为 `font-size: 16px`（iOS 安全阈值）

**11. 触摸设备会话删除按钮不可见**
- **文件**: `src/style/chat.css`
- **问题**: `.chat-session-del { opacity: 0 }` 依赖 hover 显示，触摸设备永远不可见
- **修复**: 添加 `@media (hover: none)` 使活跃会话操作按钮半透明可见

**12. 服务取消按钮 !important 滥用**
- **文件**: `src/style/pages/services.css`
- **问题**: `.service-cancel-btn` 4 条 `!important` 声明
- **修复**: 移除所有 `!important`，提高选择器特异性即可

**13. 通用按钮 !important 滥用**
- **文件**: `src/style/pages/services.css`
- **问题**: `.btn-icon` 使用 `!important` 覆盖按钮颜色
- **修复**: 移除 `!important`，使用 `color: inherit` + 正确的选择器优先级

**14. 危险按钮 hover 硬编码颜色**
- **文件**: `src/style/pages/services.css`
- **问题**: `.btn-icon.danger:hover { border-color: #ef4444 }` 硬编码
- **修复**: 使用语义变量 `var(--error)` + `var(--error-muted)`

**15. 聊天快捷指令面板 monospace 字体**
- **文件**: `src/style/chat.css`
- **问题**: `.cmd-name` 使用硬编码 `'SF Mono', monospace`
- **修复**: 改为 `var(--font-mono)` 使用设计系统变量

- **建议**: 确保所有 `.card` 基类包含 `transition: ... transform var(--ease-normal)`
