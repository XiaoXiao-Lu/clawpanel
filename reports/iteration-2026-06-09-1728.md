## 修改说明
- 将 `parseProxyBody()` 导出供测试覆盖，并增强代理响应解析：
  - 支持 `{ ok, status, body: { choices: ... } }` 这类对象型 body 包装。
  - 保持成功响应必须是合法 JSON，避免吞掉协议错误。
  - 对 `ok:false` 的 JSON 或纯文本错误体统一抛出 `API error <status>: <message>`。
- 新增专家团运行器测试，覆盖对象型 body、JSON 错误体和纯文本错误体三种情况。
