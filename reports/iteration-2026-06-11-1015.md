# Iteration 2026-06-11 (about 页版本 XSS 加固)

## 目标
- 在干净文件中挑一个可隔离的安全目标落地。
- 收敛「关于」页把远程版本字符串未转义注入 innerHTML 的 XSS 风险。

## 地毯式审查摘要
- 起始 HEAD：9dd5143e（上一轮日志加固）。工作区仍是上几轮遗留的大范围脏改（27 文件）。
- 基线：npm test 557/557 通过，npm run build 通过。
- 扫描 innerHTML + 模板插值，发现 src/pages/about.js（干净文件，不在脏改集合）存在真实 XSS 面：
  - 版本下拉项 `<option value="${v}">${v}` 直接拼接 npm registry 返回的版本 key（list_openclaw_versions → 远程 registry 的 versions 对象 key）。
  - 升级/降级提示 hintEl.innerHTML 直接插值 currentVersion.current / targetVer。
  - stat-card 版本值、policyAhead / aheadOfRecommended / recommendedStable / latestUpstream / newVersionAvailable 等 t() 参数（version.current/recommended/latest）经无转义的 t() 后注入 innerHTML。
  - i18n 的 t() 仅做纯字符串 replace，不转义，故参数值同样是注入向量。
- 后端 src-tauri/src/commands/config.rs::list_openclaw_versions 证实版本来自远程 registry JSON 的对象 key，registry 可被篡改/中间人，构成实际 XSS 触发路径。

## 修改
- src/pages/about.js：
  - 版本 `<option>` 的 value 用本地 escapeAttr、文本用 escapeHtml。
  - 三处升级提示 hintEl.innerHTML 的 currentVersion.current / targetVer 改为 escapeHtml。
  - stat-card 版本值与 policyAhead/aheadOfRecommended/recommendedStable/latestUpstream/newVersionAvailable 的版本参数全部 escapeHtml。
  - 复用文件内已有的本地 escapeAttr（未新增 import，避免与既有定义重复声明）。
  - 静态 i18n 文案、build 常量 __APP_VERSION__、Tauri getVersion()（受信）保持不变；URL 仍走既有 safeExternalHref + escapeAttr。
- 新增 tests/about-version-xss.test.js：断言 option 双重转义、innerHTML 块不再出现裸版本插值、escapeHtml 已 import 且本地 escapeAttr 存在。
