## 🟡 布局问题与建议

### 1. CSS 死代码 — 两套模型行样式共存

- **文件**: `src/style/pages/models.css`
- **问题**: 第 133-267 行定义了 `.model-row*` 样式体系，而第 1487-1537 行又定义了 `.model-item*` 样式体系。JS 实际使用 `.model-item*`，`model-row*` 已无引用但未被清理。
- **建议**: 删除第 133-267 行 `.model-row*` 整个区块（约 130 行死代码）

### 2. qtcool 推广横幅完全使用内联样式

- **文件**: `src/pages/models.js:40-45`
- **代码**:
  ```js
  <div id="qtcool-body" style="display:none;padding:8px 12px;margin-bottom:12px;border:1px solid var(--border-primary);border-radius:var(--radius-md);background:var(--bg-secondary);font-size:12px">
  <input ... style="font-size:12px;padding:5px 8px;flex:1;min-width:160px">
  ```
- **问题**: 完全内联样式，不可维护，且折叠用 `style.display=''`/`'none'` 生硬切换
- **建议**: 提取为 `.models-qtcool-body` / `.models-qtcool-body--hidden` CSS class

### 3. fallback-waterfall-toggle 使用 `style="display:none/block"` 切换

- **文件**: `src/pages/models.js:641-645`
  ```js
  container.style.display = 'none'
  container.style.display = 'block'
  ```
- **问题**: JS 直接操作 style，无过渡动画，同页面存在 `style="margin-bottom:20px"`
- **建议**: 改用 CSS class `.waterfall-hidden` 配合 `max-height` 动画

### 4. 控制台 (models-control-console) 响应式不足

- **文件**: `src/style/pages/models.css:342-352`
- **问题**: 控制台固定 `padding: var(--space-xl)` + `box-shadow: var(--shadow-md)`。当窗口缩小到 640px 以下时，`.models-console-header` 和 `.models-switch-row` 会换行，但内部的按钮组 (`.models-switch-actions` + `.models-route-presets`) 没有足够的折叠或换行策略，按钮文字可能溢出。
- **建议**: 
  - `.models-switch-actions` 在小屏时按钮宽度 100%
  - `.models-route-presets` 添加 `overflow-x: auto` 水平滚动

### 5. Provider card 按钮组溢出风险

- **文件**: `src/pages/models.js:967-972`
  ```js
  <div class="provider-card__actions">
    <button>↻</button> ← 文本字符而非 SVG
    <button>+</button>
    <button>···</button>
    <button>trash icon</button>
  </div>
  ```
- **问题**: 
  - 前三个按钮使用文本符号 (↻ + ···) 而非 SVG 图标，与 skill 要求冲突
  - 当 Provider 名称很长时，按钮组可能溢出右侧
- **建议**: 
  - 用 `icon('refresh-cw', 12)` 替代 `↻`，用 `icon('plus', 12)` 替代 `+`，用 `icon('more-horizontal', 12)` 替代 `···`
  - 给 `.provider-card__actions` 添加 ``flex-shrink: 0``

### 6. fallback-workbench 网格比例偏科

- **文件**: `src/style/pages/models.css:610-614`
  ```css
  grid-template-columns: minmax(0, 1fr) minmax(320px, 0.92fr);
  ```
- **问题**: 第二列固定 min 320px，在 680-800px 窗口宽度下右侧候选池会过度挤压左侧激活链
- **建议**: 在 800px 断点改为单列（已有 `.fallback-workbench` 的 980px 响应式，但改为 860px 更合适）

### 7. 文件体积过大

- `models.js`: **2378 行** — 所有渲染/逻辑/事件绑定全在一个文件中
- `models.css`: **1581 行** — 包含 models + connectors + combobox 三套独立 UI
- **建议**: 按职责拆分为 `models-provider.js` / `models-waterfall.js` / `models-console.js` 或至少提取 Connectors 相关 CSS 到独立文件

### 8. models.css 旧变量名大量使用

- 文件中大量使用旧别名 `--border-primary`、`--surface-raised`、`--text-tertiary` 等。虽然它们通过 backward-compat 映射正常工作，但新代码应直接使用 `--border-1`、`--bg-elevated`、`--text-3` 以保持一致性。

### 9. 搜索无高亮

- `renderModelRows` 函数接受 `search` 参数，但模型名称没有匹配高亮，搜索结果不够直观
- **建议**: 当 `search` 非空时，用 `<mark>` 或 `.model-highlight` 包裹匹配文本

### 10. 折叠交互不一致

- provider-card: 使用 `[hidden]` + CSS transition ✅
- qtcool promo: 使用 `style.display` ❌
- fallback candidates: 使用 JS toggle + `renderDefaultBar()` ❌
- **建议**: 统一折叠模式为 CSS class 驱动
