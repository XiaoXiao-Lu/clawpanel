# ChatCraw 持续迭代报告

## 本轮目标
- 修复启动登录覆盖层的一个小型安全缺口：默认密码回填到 `<input value="...">` 属性前未做 HTML 转义。
- 为该安全边界补充回归测试，避免后续登录页模板调整时重新引入属性注入风险。

## 发现的问题
- `showLoginOverlay(defaultPw)` 会把桌面端或 Web 模式提供的默认密码直接插入登录框 `value` 属性。
- 正常默认密码通常是固定值，但该值来自配置或后端响应，属于外部输入；如果包含引号或标签片段，会破坏 HTML 属性边界。

## 已修改文件
- `src/main.js`
- `tests/login-overlay-security.test.js`

## 修改说明
- 在 `showLoginOverlay` 内新增 `defaultPwValue`，使用现有 `escapeHtml(defaultPw)` 对默认密码做属性安全转义后再拼入登录框。
- 新增 `tests/login-overlay-security.test.js`，静态锁定登录默认密码回填必须使用转义后的 `defaultPwValue`，并禁止恢复为原始 `defaultPw` 拼接。
