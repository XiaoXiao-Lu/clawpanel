# Iteration 2026-06-10 16:10

## 目标
- 修复专家团执行流程类 Markdown 文档在助手专家团交付/详情里的显示问题。
- 保持当前 `main` 开发，只提交本轮可安全区分的改动。

## 问题
- 轻量 Markdown 渲染器只支持 1-3 级标题，缺少常见文档语法支持。
- 专家团输出类似 `执行流程.md` 时，`####` 标题、任务列表、引用和分割线容易按普通文本显示，交付内容不够像正式文档。
- 专家团详情/交付容器没有完整继承普通消息 Markdown 的文档样式。

## 修改
- 扩展 `renderMarkdown()`：支持 1-6 级标题、分割线、引用块和任务列表。
- 保持安全策略：链接、图片、代码块和原始 HTML 仍按既有转义/协议白名单处理。
- 为普通助手消息和专家团交付/详情内容补齐标题、引用、分割线、任务列表和表格样式。
- 新增回归测试覆盖“专家团执行流程.md”典型语法。

- 渲染器仍是轻量 Markdown，不等同完整 CommonMark；嵌套引用、嵌套列表等复杂文档语法后续仍可继续增强。
- 构建后 CSS 体积仍偏大，本轮未处理。
