## 🔴 已修复 — P0 级别问题 (6 项)

### 1. layout.css — 重复 `.nav-section` 选择器
- **位置**: `src/style/layout.css` 第 286 行和第 499 行
- **问题**: 同一选择器定义了两次，浪费维护成本
- **修复**: 删除第 499 行的重复定义，保留第 286 行的规范定义

### 2. dashboard.css — 重复 `.quick-actions` 选择器
- **位置**: `src/style/pages/dashboard.css` 第 29 行和第 417 行
- **问题**: 两处定义使用不同的 `gap` 和 `margin-bottom`，后者覆盖前者
- **修复**: 合并为一处定义，使用统一的 `gap: var(--space-3)` 和 `margin-bottom: var(--space-5)`

### 3. dashboard.css — 重复 `.dashboard-log-viewer` 选择器
- **位置**: `src/style/pages/dashboard.css` 第 37 行和第 640 行
- **问题**: 两处完全相同的 `max-height: 300px` 定义
- **修复**: 删除第 640 行的重复定义

### 4. channels.css — 冲突的 `.toggle-switch` 定义
- **位置**: `src/style/pages/channels.css` 第 781-816 行 vs `src/style/components.css` 第 570-615 行
- **问题**: channels.css 定义了简化版 toggle-switch，缺少 focus-visible、disabled 状态，且使用硬编码 `0.2s` 过渡
- **修复**: 删除 channels.css 中的重复定义，统一使用 components.css 的完整版本

### 5. models.css — 重复 `model-row-highlight` 关键帧
- **位置**: `src/style/pages/models.css` 第 797 行和第 1608 行
- **问题**: 完全相同的关键帧定义了两次
- **修复**: 删除第 1608 行的重复定义

### 6. agents.css — 61+ 处硬编码颜色值
- **位置**: `src/style/agents.css` 全文
- **问题**: 大量 `#2dd4bf`、`#22c55e`、`#f59e0b`、`#ef4444`、`#f8fafc`、`#111827` 等硬编码颜色，不随主题切换
- **修复**:
  - 在 `variables.css` 新增 10 个 agent 状态颜色变量（含亮色/暗色主题适配）
  - 替换 agents.css 中所有硬编码颜色为 CSS 变量
  - 修复 `color-mix()` 中的 `#0f172a`、`#ffffff`、`#111827` 等硬编码引用

## 🟡 已修复 — P1 级别问题 (5 项)

### 7. chat.css — 17 处硬编码过渡时间
- **位置**: `src/style/chat.css` 多处
- **问题**: `transition: ... 0.2s ease`、`0.18s ease`、`0.15s`、`0.12s`、`0.1s` 等硬编码值，不使用设计令牌
- **修复**: 统一替换为 `var(--ease-fast)` (150ms)

### 8. chat.css — 硬编码 `#fff` 颜色
- **位置**: `src/style/chat.css` 第 1942 行
- **问题**: `.ha-toggle-track::after` 使用 `background: #fff`
- **修复**: 替换为 `var(--text-inverse)`

### 9. 响应式断点不一致 — 760px vs 768px
- **位置**: `chat.css`、`dashboard.css`、`expert-teams.css`、`misc.css`
- **问题**: 4 个文件使用 `max-width: 760px`，而主断点是 `768px`，导致布局在 760-768px 区间不一致
- **修复**: 统一为 `max-width: 768px`

### 10. layout.css — 侧边栏链接缺少 `cursor: pointer`
- **位置**: `src/style/layout.css` 第 410 行
- **问题**: `.sidebar-link` 有 `color` 和 `transition` 但缺少显式 `cursor: pointer`
- **修复**: 添加 `cursor: pointer`

### 11. variables.css — 新增 agent 状态颜色变量
- **位置**: `src/style/variables.css`
- **问题**: agent 状态颜色（idle/working/blocked/error）没有对应的 CSS 变量
- **修复**: 新增 `--agent-idle`、`--agent-working`、`--agent-blocked`、`--agent-error` 及其文本色变体，含亮色/暗色主题适配

## 🔵 保留 — 可选优化建议 (8 项)

### 12. assistant.css 体积过大 (90KB)
- **建议**: 拆分为 `assistant-core.css` + `assistant-features.css`，减少单文件维护负担

### 13. 57 处 `!important` 声明
- **分布**: assistant.css (21), chat.css (8), layout.css (5), models.css (5), expert-teams.css (5), misc.css (5), agents.css (2), polish.css (2)
- **建议**: 使用 CSS Cascade Layers (`@layer`) 或更具体的选择器替代 `!important`

### 14. 50+ 处 JS 内联样式
- **位置**: main.js, sidebar.js, ai-drawer.js, channels.js, toast.js 等
- **问题**: `style.xxx =` 直接赋值，不利于主题切换和维护
- **建议**: 将状态颜色迁移到 CSS class，显示/隐藏使用 `classList.toggle()`

### 15. 部分语义 HTML 缺失
- **位置**: `index.html`
- **问题**: 缺少 `<nav>`、`<section>`、`<header>`、`<footer>` 等语义标签
- **建议**: 为侧边栏添加 `<nav aria-label="Main navigation">`，为内容区添加 `<section>`

### 16. 表单无障碍待完善
- **问题**: 部分表单控件缺少 `for/id` 关联、`aria-describedby`、`aria-required`
- **建议**: 为所有 `<label>` 添加 `for` 属性，关联对应的 `<input id>`

### 17. 部分图标按钮缺少 `aria-label`
- **建议**: 为所有纯图标按钮（如 toast 关闭、侧边栏按钮）添加 `aria-label`

### 18. 搜索结果计数未播报
- **位置**: sidebar.js 搜索
- **问题**: 侧边栏搜索过滤结果未通过 `aria-live` 播报
- **建议**: 添加 `aria-live="polite"` 播报搜索结果数量

### 19. `--radius-md` 和 `--radius-lg` 值相同
- **位置**: `variables.css` 第 142-143 行
- **问题**: `--radius-md: 8px` 和 `--radius-lg: 8px` 值完全相同，语义区分无意义
- **建议**: 将 `--radius-lg` 调整为 `12px` 以体现层级差异
