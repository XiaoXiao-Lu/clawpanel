## 本轮修复

### P0: 添加 light 主题缺失的 --accent-* 变量别名

**文件**: [variables.css:543-549](src/style/variables.css#L543-L549)

**问题**: light 主题 (`[data-theme="light"]`) 块中缺少 `--accent` / `--accent-muted` / `--accent-bg` / `--accent-border` / `--accent-subtle` 等变量别名定义，导致 `assistant.css` 多处样式（如 `.ast-expert-team-run`、`.ast-guide` 等）在浅色主题下无法正确解析。

**修复**: 在 light 主题块中添加完整别名集，与 dark 主题保持一致：
```css
--accent:                var(--aether-primary);
--accent-hover:          var(--aether-primary-light);
--accent-muted:          var(--aether-primary-muted);
--accent-bg:            var(--aether-primary-faint);
--accent-border:         rgba(91, 78, 232, 0.24);
--accent-subtle:         var(--aether-primary-faint);
```

### P1: 修复 assistant.css 硬编码回退颜色值

**文件**: [assistant.css:1915](src/style/assistant.css#L1915)

**问题**: 回退值 `rgba(var(--accent-rgb, 139,92,246), 0.08)` 中使用了与设计系统不一致的颜色值（139,92,246 = #8B5CF6）。

**修复**: 统一回退值为 `rgba(var(--accent-rgb, 91,78,232), 0.08)`，与 variables.css 中 `--accent-rgb` 定义一致。

### P1: Expert Teams 详情面板 ARIA tablist 可访问性增强

**文件**: [assistant.js:4451-4467](src/pages/assistant.js#L4451-L4467)

**问题**: 详情面板使用 CSS-only tab 切换但缺少 ARIA role，导致屏幕阅读器无法识别 tab 结构。

**修复**:
1. 给 `.ast-expert-detail-body` 添加 `role="tablist"` 和 `aria-label`
2. 给每个 label 添加 `role="tab"`、`aria-selected`、`tabindex`
3. 给每个 `.ast-expert-detail-content` 添加 `role="tabpanel"`、`aria-labelledby`，非活跃 tabpanel 添加 `hidden`
4. 添加 CSS 规则同步 `aria-selected` 状态和 `[hidden]` 隐藏
