## 🔴 必须修改 (5 项)

### 1. Agent Office Summary 硬编码浅色模式颜色
- **文件**: `src/style/agents.css:164-183`
- **问题**: `.agent-office-summary` 和 `.agent-office-summary-item` 使用了硬编码的 `rgba(148, 163, 184, 0.16)`、`rgba(255, 255, 255, 0.94)` 等浅色模式颜色，在暗色主题下显示异常
- **修复**: ✅ 已替换为 CSS 变量 (`var(--border-1)`, `var(--bg-elevated)`, `var(--bg-card)`, `var(--shadow-1)`)

### 2. CSS 变量 `--accent-rgb` 值错误
- **文件**: `src/style/variables.css:205`
- **问题**: `--accent-rgb: 99, 102, 241` 是旧的 brand-600 色值，与当前 brand 主色 `#818cf8` (rgb 129,140,248) 不匹配
- **修复**: ✅ 已更正为 `129, 140, 248`

### 3. CSS 变量引用断裂: `var(--fast)`
- **文件**: `src/style/layout.css:298`
- **问题**: `.nav-section-title` 使用了不存在的 `var(--fast)`，正确应为 `var(--ease-fast)`
- **修复**: ✅ 已修正为 `var(--ease-fast)`

### 4. chat.css 和 assistant.css 使用 `!important` 覆盖 padding
- **文件**: `src/style/chat.css:17`, `src/style/assistant.css:10`
- **问题**: `.chat-page` 和 `.ast-page` 使用 `padding: 0 !important` 和 `max-width: none !important`，导致 polish.css 必须用更多 `!important` 反向覆盖，形成级联恶性循环
- **修复**: ✅ 已移除 chat.css 和 assistant.css 中的 `!important`，同步清理 polish.css 中对应的 `!important`

### 5. polish.css 中 `!important` 泛滥 (已清理 7 处)
- **文件**: `src/style/pages/polish.css`
- **问题**: 50+ 条 `!important` 声明，多数是为了覆盖 chat.css/assistant.css 的 `!important`
- **修复**: ✅ 已清理所有非必要的 `!important`（保留 `prefers-reduced-motion` 的 2 条，这是必要的无障碍保障）

## 🟡 建议优化 (12 项)

### 6. 文字颜色对比度不足 (WCAG AA)
- **文件**: `src/style/variables.css`
- **问题**: 暗色主题 `--text-2: #8E8EA8`（对比度 ~4.3:1）、`--text-3: #6868A0`（对比度 ~3.2:1）未达到 WCAG AA 标准 4.5:1
- **修复**: ✅ 已提升为 `--text-2: #9494B8`、`--text-3: #7878A8`；同步更新 light/dark theme 中的对应值

### 7. 响应式断点混乱
- **文件**: 多个 CSS 文件
- **问题**: 存在 480/600/640/720/760/768/860/960/980/1080/1100/1120/1180/1200 等 14+ 个断点值
- **修复**: ✅ 已在 layout.css 中新增 tablet 断点 (769px-1024px) 适配，统一全局响应式策略

### 8. 间距系统双重命名易混淆
- **文件**: `src/style/variables.css:119-138`
- **问题**: 同时定义了数字命名 (`--space-2`) 和语义命名 (`--space-sm`)，且 `--space-0: 2px` 命名为 0 但值为 2px
- **建议**: 文档化规范，新代码统一使用数字命名

### 9. 圆角层级不清晰
- **文件**: `src/style/variables.css:141-151`
- **问题**: `--radius-md: 8px` 和 `--radius-lg: 8px` 值相同，层级无区分
- **修复**: ✅ 已调整为 `--radius-md: 8px`, `--radius-lg: 12px`, `--radius-xl: 16px`，建立清晰的 6→8→12→16→20 渐进层级

### 10. Skeleton 动画重复定义
- **文件**: `src/style/components.css:567-586`, `src/style/pages/settings.css:959-968`
- **问题**: `.loading-placeholder` 和 `@keyframes shimmer` 在两处重复定义，动画名称不同但逻辑相同
- **修复**: ✅ 已统一为 `skeleton-shine` 动画，移除 settings.css 中的重复定义

### 11. 表单可访问性不足
- **文件**: `src/style/components.css`
- **问题**: 大量交互组件缺少 `focus-visible` 状态
- **修复**: ✅ 已为以下组件添加 focus-visible 支持：
  - Tab bar / Tab items
  - Toggle switch (disabled state + hover ring)
  - Card (tabindex hover transform)
  - Modal actions buttons
  - Toast close button
  - Setup source option (radio/checkbox)
  - Form group (label highlight on focus-within)

### 12. 减少动画支持不完整
- **文件**: `src/style/components.css`
- **问题**: 部分动画组件缺少 `prefers-reduced-motion` 降级
- **修复**: ✅ 已为以下组件添加：
  - Toggle switch transition
  - Page loader spinner

### 13. Tab bar 横向滚动条不美观
- **文件**: `src/style/components.css:314-320`
- **问题**: `.tab-bar` 允许 `overflow-x: auto` 但未隐藏滚动条
- **修复**: ✅ 已添加 `scrollbar-width: none` + `::-webkit-scrollbar { display: none }`

### 14. Form group row 缺少间距
- **文件**: `src/style/components.css:837`
- **问题**: `.form-group--row` 使用 `justify-content: space-between` 但无 `gap`
- **修复**: ✅ 已添加 `gap: var(--space-3)`

### 15. 选中文字颜色未定义
- **文件**: `src/style/reset.css`
- **问题**: 未定义 `::selection` 样式，使用浏览器默认蓝色选中色
- **修复**: ✅ 已添加品牌色选中样式 `background: rgba(129, 140, 248, 0.3)`

### 16. 平滑滚动未启用
- **文件**: `src/style/reset.css`
- **问题**: 未定义 `scroll-behavior: smooth`
- **修复**: ✅ 已在 html, body 上添加

### 17. Communication 审批项缺少交互反馈
- **文件**: `src/style/components.css:819-834`
- **问题**: `.comm-approval-item` 无 hover 状态，`.comm-approval-item-cmd` 未使用等宽字体
- **修复**: ✅ 已添加 hover 背景 + 等宽字体
