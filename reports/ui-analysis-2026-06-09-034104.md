## 📊 执行摘要

基于对 ClawPanel 项目的全面 UI 审查，发现 **23 个主要问题** 和 **15 个优化机会**。整体设计系统（Precision Noir）基础扎实，但在一致性、可访问性和交互细节方面存在改进空间。

**总体评分**: 72/100（中等偏上，有明显优化空间）

## 🔴 必须修改 (11项)

### 1. CSS 变量命名不一致
**位置**: `src/style/variables.css`  
**问题**: 混用新旧命名约定，导致维护困难
```css
/* 新命名 */
--brand: #818cf8;
--brand-400: #a5b4fc;

/* 旧命名（遗留） */
--accent: var(--brand);
--accent-hover: var(--brand-400);
--primary: var(--brand);
--primary-hover: var(--brand-400);
```
**影响**: 开发者困惑，样式覆盖困难  
**修复**: 统一使用 `--brand-*` 命名，删除所有 `--accent-*` 和 `--primary-*` 旧变量

### 2. 间距系统混乱
**位置**: `src/style/variables.css` 第119-138行  
**问题**: 间距变量有数字和语义两种命名，容易混淆
```css
--space-1: 4px;
--space-xs: var(--space-1);

--space-2: 8px;
--space-sm: var(--space-2);

--space-3: 12px;
--space-md: var(--space-3);
```
**影响**: 使用时不知道该用数字还是语义命名  
**修复**: 保留语义命名（xs/sm/md/lg/xl），删除数字命名，或至少添加注释说明推荐用法

### 3. 按钮尺寸不统一
**位置**: `src/style/components.css`  
**问题**: 按钮 min-height 值不一致
```css
.btn { min-height: 38px; }
.btn-sm { min-height: 32px; }
.btn-xs { min-height: 24px; }
```
**移动端要求**: `@media (max-width: 768px)` 设置 `min-height: 44px`，但未应用到所有按钮变体  
**修复**: 确保所有按钮在移动端都有足够的触摸目标

### 4. 表单输入框缺少标签关联
**位置**: 多个页面组件  
**问题**: 大量 `<input>` 没有 `id` 或关联的 `<label>`，仅依赖 `placeholder`  
**可访问性风险**: 屏幕阅读器无法正确识别输入框用途  
**修复**: 为所有表单控件添加 `id` 和关联的 `<label>`

### 5. 模态框焦点管理不完整
**位置**: `src/components/modal.js`  
**问题**: `showModal()` 函数没有实现焦点陷阱
```javascript
// showConfirm 有 trapFocus
trapFocus(overlay, modalEl)

// showModal 没有
const modalEl = overlay.querySelector('.modal')
// 缺少 trapFocus(overlay, modalEl)
```
**影响**: 用户可以 Tab 到模态框外部，体验差  
**修复**: 为 `showModal()` 添加 `trapFocus()` 调用

### 6. Toast 通知位置在桌面模式下不正确
**位置**: `src/style/components.css` 第372-383行  
**问题**: 桌面模式下 Toast 顶部偏移计算有问题
```css
.toast-container {
  top: var(--space-4); /* 默认 16px */
}
body.has-desktop-chrome .toast-container {
  top: calc(36px + var(--space-4)); /* 36px + 16px = 52px */
}
```
**问题**: 但 `body.has-desktop-chrome` 的条件判断可能在 Tauri 环境中不触发  
**修复**: 确保 Tauri 桌面模式正确添加 `has-desktop-chrome` 类

### 7. 侧边栏折叠状态缺少过渡动画
**位置**: `src/style/layout.css` 第121-147行  
**问题**: 折叠/展开没有平滑过渡
```css
#sidebar {
  transition: width var(--ease-normal);
}
/* 但折叠态的内容隐藏是瞬间的 */
#sidebar.sidebar-collapsed .sidebar-title,
#sidebar.sidebar-collapsed .nav-item span {
  display: none; /* 没有过渡 */
}
```
**影响**: 视觉跳跃感  
**修复**: 使用 `opacity` 过渡替代 `display: none`

### 8. 卡片悬停效果过于激进
**位置**: `src/style/pages/polish.css` 第26-42行  
**问题**: 所有卡片悬停都有 `transform: translateY(-1px)`
```css
.card:hover,
.stat-card:hover,
.overview-card:hover,
.service-card:hover,
/* ... 等等 */
{
  transform: translateY(-1px);
}
```
**影响**: 在密集布局中造成视觉抖动  
**修复**: 仅对可点击卡片保留上移效果，静态卡片只改变边框和阴影

### 9. 响应式断点不完整
**位置**: `src/style/layout.css`  
**问题**: 缺少中等屏幕断点（768px-1024px）
```css
@media (max-width: 768px) { /* 移动端 */
  .mobile-topbar { display: flex; }
  /* ... */
}

@media (max-width: 480px) { /* 小屏移动 */
  .page { padding: var(--space-3); }
}
/* 缺少 768px-1024px 的平板适配 */
```
**影响**: 平板用户体验差  
**修复**: 添加 `@media (max-width: 1024px)` 断点

### 10. 颜色对比度不足
**位置**: 多处  
**问题**: 某些文本颜色对比度低于 WCAG AA 标准（4.5:1）
```css
/* 示例 */
--text-3: #6868A0; /* 在深色背景上对比度约 3.8:1 */
--text-tertiary: var(--text-3);
```
**影响**: 视力障碍用户难以阅读  
**修复**: 调整 `--text-3` 为更亮的颜色（如 `#7A7AB5`）

### 11. 缺少加载状态骨架屏规范
**位置**: 多个页面  
**问题**: 骨架屏样式不一致
```css
/* components.css */
.skeleton { background: linear-gradient(90deg, var(--bg-sunken) 25%, var(--bg-hover) 50%, var(--bg-sunken) 75%); }

/* dashboard.css */
.loading-placeholder { background: linear-gradient(90deg, var(--bg-sunken) 25%, var(--bg-hover) 50%, var(--bg-sunken) 75%); }
```
**修复**: 统一使用 `.skeleton` 类，删除重复定义

## 🟡 建议优化 (8项)

### 12. 按钮 hover 状态缺少焦点样式
**位置**: `src/style/components.css`  
**问题**: 按钮只有 `:focus-visible` 样式，缺少 `:focus` 样式
```css
.btn:focus-visible {
  outline: 2px solid var(--brand);
  outline-offset: 2px;
  box-shadow: var(--shadow-glow);
}
/* 缺少 :focus 样式（用于键盘和鼠标混合操作） */
```
**建议**: 添加 `:focus` 样式，确保所有交互方式都有视觉反馈

### 13. 表单验证状态样式不完整
**位置**: `src/style/components.css` 第249-291行  
**问题**: 验证状态只有边框颜色变化，缺少图标反馈
```css
.input--error { border-color: var(--error); }
.input--success { border-color: var(--success-muted); }
/* 缺少验证图标 */
```
**建议**: 在输入框右侧添加验证状态图标（✓/✗）

### 14. 移动端触摸反馈不足
**位置**: `src/style/layout.css` 第919-924行  
**问题**: 触摸目标大小足够，但缺少视觉反馈
```css
@media (max-width: 768px) {
  .nav-item { min-height: 44px; }
  .btn:not(.btn-xs):not(.btn-sm) { min-height: 44px; }
  /* 缺少 :active 状态样式 */
}
```
**建议**: 添加触摸设备的 `:active` 状态样式

### 15. 暗色主题对比度可优化
**位置**: `src/style/variables.css` 第274-303行  
**问题**: 暗色主题的卡片背景层次不够分明
```css
[data-theme="dark"] {
  --bg-base: #0E0E1C;
  --bg-elevated: #151528;
  --bg-sunken: #1A1A30;
  /* 差异较小，在某些屏幕上难以区分 */
}
```
**建议**: 增加背景层级差异（如 `--bg-elevated: #1A1A32`）

### 16. 动画缺少 prefers-reduced-motion 支持
**位置**: `src/style/layout.css` 第935-941行  
**问题**: 虽然有全局减少动画的媒体查询，但部分动画未遵循
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
/* 但某些组件直接在 style 属性中定义了动画 */
```
**建议**: 确保所有动画都尊重 `prefers-reduced-motion`

### 17. 侧边栏搜索框缺少防抖
**位置**: `src/components/sidebar.js` 第91-92行  
**问题**: 搜索框可能每次按键都触发过滤
```html
<input class="sidebar-search-input" id="sidebar-search" type="text" placeholder="搜索..." autocomplete="off">
```
**建议**: 添加 300ms 防抖，减少不必要的过滤计算

### 18. 模态框关闭动画缺失
**位置**: `src/components/modal.js`  
**问题**: 模态框只有打开动画，关闭时瞬间消失
```javascript
// 打开有动画
animation: modalIn 200ms var(--ease-out);

// 关闭直接移除
overlay.remove() // 没有关闭动画
```
**建议**: 添加淡出动画

### 19. 表格在小屏幕下缺少横向滚动
**位置**: 多个页面  
**问题**: 数据表格在小屏幕下可能溢出
```css
/* 缺少表格容器样式 */
.table-container {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}
```
**建议**: 为所有表格添加横向滚动容器

### 22. 空状态设计可增强

### 23. 工具提示（Tooltip）缺少延迟
