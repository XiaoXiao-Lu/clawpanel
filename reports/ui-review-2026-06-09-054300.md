## 🔴 必须修改 (已修复 6 项)

### 1. CSS 变量 `--font-size-md` 未定义
- **文件**: `variables.css` → `reset.css`
- **问题**: `reset.css` 引用 `var(--font-size-md)` 但该变量从未在 `:root` 中定义，导致 `font-size` 回退到浏览器默认值
- **修复**: 在 `variables.css` 中添加 `--font-size-md: var(--text-base);`，同时将 `reset.css` 中的引用改为 `var(--text-base)`

### 2. CSS 变量 `--accent-rgb` 和 `--accent-border` 缺失
- **文件**: `variables.css`
- **问题**: `chat.css`、`assistant.css` 等多处引用 `var(--accent-rgb)` 和 `var(--accent-border)` 但从未定义，导致 `color-mix()` 和 `rgba()` 计算失败
- **修复**: 在 `:root`、light theme、dark theme 三个作用域中分别添加对应的 RGB 值和边框色

### 3. CSS 变量 `--brand-indigo` 不存在
- **文件**: `assistant.css:763`
- **问题**: `.ast-model-config-btn` 引用 `var(--brand-indigo)` 但该变量从未定义
- **修复**: 替换为 `var(--brand)`

### 4. 重复 CSS 选择器定义
- **文件**: `assistant.css`
- **问题**: `.ast-msg-bubble-ai ul/ol`、`.ast-msg-bubble-ai li`、`.ast-msg-bubble-ai strong`、`.ast-msg-bubble-ai h1/h2/h3` 各有两套定义，后面的覆盖前面的，造成维护混乱
- **修复**: 合并为单一定义，保留正确的值

### 5. 重复 `.status-dot` 定义
- **文件**: `chat.css` vs `components.css`
- **问题**: `chat.css` 用硬编码颜色重新定义了 `.status-dot`，覆盖了 `components.css` 中使用 CSS 变量的版本
- **修复**: 移除 `chat.css` 中的重复定义，统一使用 `components.css` 的版本

### 6. 聊天输入框 `font-size: 16px` 硬编码
- **文件**: `chat.css:899`
- **问题**: 聊天输入 textarea 使用硬编码 `font-size: 16px`，在 iOS 上会阻止页面缩放（iOS Safari 在 font-size < 16px 时会自动缩放），且不随设计系统缩放
- **修复**: 改为 `var(--text-base)`

## 🟡 建议优化 (已修复 5 项)

### 7. 导航激活态使用未定义 `--accent` 变量
- **文件**: `layout.css:533`
- **问题**: `.nav-item.active` 的 `box-shadow` 使用 `var(--accent)` 而非 `var(--brand)`，虽然 `--accent` 是 `--brand` 的别名，但命名不一致
- **修复**: 统一使用 `var(--brand)`

### 8. 导航激活态 SVG 使用 `!important`
- **文件**: `layout.css:568`
- **问题**: `.nav-item.active svg { color: var(--brand) !important; }` 使用 `!important` 不必要，因为选择器特异性已足够
- **修复**: 移除 `!important`

### 9. 缺少大屏断点 (≥1440px)
- **文件**: `layout.css`
- **问题**: 只有 tablet (769-1024px) 和 mobile (≤768px) 断点，缺少大屏优化
- **修复**: 添加 `@media (min-width: 1440px)` 断点，将侧边栏宽度增加到 260px

### 10. 缺少小屏 (≤480px) 统计卡片适配
- **文件**: `layout.css`
- **问题**: 小屏手机上统计卡片 grid 仍尝试多列，导致卡片过窄
- **修复**: 在 480px 断点下强制 `grid-template-columns: 1fr`

### 11. 触摸设备上卡片 hover `translateY` 导致布局抖动
- **文件**: `polish.css`
- **问题**: 触摸设备上 `:hover` 触发时卡片会上移 1px，但触摸设备没有持续 hover 状态，导致闪烁
- **修复**: 在 `@media (hover: none)` 下禁用 `transform: translateY(-1px)`

## 🔵 可选优化项 (8 项，未修改)

### 12. Agent Office 硬编码浅色主题颜色
- **文件**: `agents.css`
- **问题**: `.agent-office-body`、`.agent-office-scene`、`.agent-office-diagnostics` 等使用硬编码 `#f8fafc`、`rgba(255,255,255,...)` 等浅色值
- **说明**: Agent Office 是故意设计为浅色 3D 工作空间，dark mode 已有 `[data-theme="dark"]` 覆盖。但覆盖可能不完整

### 13. `body::after` 噪点纹理 z-index 过高
- **文件**: `reset.css`
- **问题**: `body::after` 使用 `z-index: var(--z-max)` (1000)，虽然有 `pointer-events: none`，但理论上应使用更高值确保在所有浮层之上
- **说明**: 当前值已足够，因为 `--z-max` (1000) 高于所有组件 z-index

### 14. `polish.css` 中的 `!important` 清理
- **文件**: `polish.css`
- **问题**: 之前轮次已清理部分 `!important`，但仍有遗留
- **说明**: 需要逐步迁移使用 `--brand-*` 命名的组件，然后才能安全移除

### 15. 间距系统双命名混乱
- **文件**: `variables.css`
- **问题**: `--space-2: 8px` 和 `--space-sm: var(--space-2)` 并存，数字命名和语义命名混用
- **说明**: 建议逐步统一为语义命名 (xs/sm/md/lg/xl)

### 16. Hermes 引擎字体依赖网络加载
- **文件**: `hermes.css`
- **问题**: Hermes 使用 `--hm-font-serif: 'Playfair Display'` 但未通过 `@font-face` 或 Google Fonts 加载，依赖系统回退字体
- **说明**: 需要在 `index.html` 中添加对应的 `<link>` 标签

### 17. 模态框焦点陷阱不完整
- **文件**: `components.css`
- **问题**: `showModal()` 缺少完整的焦点陷阱（Tab 键不应逃出模态框）
- **说明**: 需要 JavaScript 层面的焦点管理

### 18. 表单 label 关联不完整
- **文件**: 多个页面组件
- **问题**: 部分 `<input>` 缺少 `id` 和关联的 `<label>`
- **说明**: 已在之前的轮次中为 15+ 组件添加了 `focus-within` 标签高亮

### 19. `--topbar-height` 变量缺失
- **文件**: `polish.css:500`
- **问题**: `.ast-sidebar.open` 在小屏下使用 `var(--topbar-height, 56px)` 但该变量未在 `variables.css` 中定义
- **说明**: 使用了 fallback 值 56px，功能正常但不规范
